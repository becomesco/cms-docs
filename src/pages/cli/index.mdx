# BCMS CLI

BCMS CLI helps you with things like scaffolding BCMS projects. It can help you quickly create anything BCMS-related. From plugins, over functions, to creating the whole codebase in your framework of choice.


## Plugin options {{ tag: 'POST', label: '/v1/contacts' }}

<Row>
  <Col>
    ## Plugin options {{ tag: 'POST', label: '/v1/contacts' }}

    `--plugin` options are for when you want to create a BCMS Plugin.
    <Properties>
      <Property name="create" type="string">
        Create new plugin project
      </Property>
      <Property name="bundle" type="string">
        Prepare plugin for deployment
      </Property>
      <Property name="deploy" type="string">
        Deploy plugin to your BCMS instance
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --plugin create
    ```
    </CodeGroup>
     Read more about creating [BCMS plugins here](/cloud/plugins).
  </Col>
</Row>

  <Col sticky>

  <Col>
    ## CMS options

    `--cms` options are for when you want to interact with a BCMS instance.
    <Properties>
      <Property name="create" type="boolean">
        Create a new BCMS project
      </Property>
      <Property name="bundle" type="boolean">
        Prepare project for deployment
      </Property>
      <Property name="deploy" type="boolean">
        Deploy the project to the BCMS Cloud
      </Property>
      <Property name="clone" type="boolean">
        Create a starter project and get instance data if available
      </Property>
      <Property name="pull" type="boolean">
        Pull functions, events, jobs and plugins for the current instance
      </Property>
      <Property name="create local backup of the BCMS" type="boolean">
        Create local backup of the BCMS
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --cms create
    ```
    </CodeGroup>
  </Col>
</Row>

  <Col>
    ## Data migration

    `--migration <command> <options>` is for when you want to migrate data from one BCMS instance to another.
    <Properties>
      <Property name="pull">
        Pull data from existing BCMS
      </Property>
      <Property name="transform">
        Transform data from 1 version to next version
      </Property>
      <Property name="create-fsdb">
        Create valid FSDB database with current data
      </Property>
      <Property name="push">
        Restore data to target BCMS
      </Property>
      </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --instance install
    ```
    </CodeGroup>
  </Col>
</Row>
