# BCMS CLI

BCMS CLI helps you with things like scaffolding BCMS projects. It can help you quickly create anything BCMS-related. From plugins, over functions, to creating the whole codebase in your framework of choice.

---

<Row>
  <Col>
    ## Plugin {{ label: '--plugin <command>' }}

    Plugin options are for when you want to create a BCMS Plugin.
    <Properties>
      <Property name="create">
        Create new plugin project
      </Property>
      <Property name="bundle">
        Prepare plugin for deployment
      </Property>
      <Property name="deploy">
        Deploy plugin to your BCMS instance
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --plugin create
    ```
    </CodeGroup>
     Read more about working with [BCMS plugins](/cloud/plugins).
  </Col>
</Row>

---

<Row>
  <Col>
    ## Function {{ label: '--function <name> <options>' }}

    Function options are for when you want to create a BCMS Function.
    <Properties>
      <Property name="--create" type="boolean">
        Create a function
      </Property>
      <Property name="--bundle" type="boolean">
        Make a function public
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --function myFunction --create
    ```
    </CodeGroup>
     Read more about creating [BCMS functions here](/cloud/plugins).
  </Col>
</Row>

---

<Row>
  <Col>
    ## Login {{ label: '--login <options>' }}

    Login options are for when you want to log in to a BCMS instance.
    <Properties>
      <Property name="otp" type="string">
        Use a specified OTP for login
      </Property>
      <Property name="terminal-login" type="boolean">
        Prompt login in a terminal session
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --login terminal-login
    ```
    </CodeGroup>
  </Col>
</Row>

---

<Row>
  <Col>
    ## CMS {{ label: '--cms <command>' }}

    CMS options are for when you want to interact with a BCMS instance.
    <Properties>
      <Property name="create">
        Create a new BCMS project
      </Property>
      <Property name="bundle">
        Prepare project for deployment
      </Property>
      <Property name="deploy">
        Deploy the project to the BCMS Cloud
      </Property>
      <Property name="clone">
        Create a starter project and get instance data if available
      </Property>
      <Property name="pull">
        Pull functions, events, jobs and plugins for the current instance
      </Property>
      <Property name="backup">
        Create local backup of the BCMS
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --cms create
    ```
    </CodeGroup>
  </Col>
</Row>

---

<Row>
  <Col>
    ## Instance {{ label: '--instance <id> <options>' }}

    Instance options are for when you want to install an instance on the current server.
    <Properties>
      <Property name="--install">
        Install instance on the current server
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms --instance yourBcmsInstanceId install
    ```
    </CodeGroup>
  </Col>
</Row>

---

<Row>
  <Col>
    ## Most {{ label: '--most <command>' }}

    Most commands are for when you want to work with data from BCMS instance
    <Properties>
      <Property name="pull-content">
        Pull content from the BCMS instance
      </Property>
      <Property name="pull-media">
        Pull media from the BCMS
      </Property>
      <Property name="pull-types">
        Create types based on the data from the BCMS instance
      </Property>
      <Property name="all">
        Run all commands
      </Property>
    </Properties>
  </Col>
  <Col sticky>
    <CodeGroup title="Example command">
    ```bash {{ title: 'cURL' }}
    bcms pull-content
    ```
    </CodeGroup>
  </Col>
</Row>
